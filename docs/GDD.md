## 3. 游戏系统详细说明 (Game Systems)

### 3.1 探索与地图系统 (Exploration & Map)

- 地图结构： 游戏总共由 N 层塔组成 。
- 网格机制： 每层地图为 a×a 的网格 。
- 实体占位： 玩家、怪物、障碍物均占据 1 个网格。
- 移动规则： 玩家仅可进行上、下、左、右四向移动，每次移动 1 格。无法穿过障碍物网格 。

### 3.2 战斗系统 (Combat System - ATB机制)

- 触发条件： 玩家移动到与怪物相邻或重合的格子时，触发战斗 。
- 核心机制 (ATB)： 玩家与怪物共享一个进度条界面（区间为 0→1） 。
    - 双方根据各自的【速度 (SPD)】属性，决定指针从 0 走向 1 的填充速率 。
    - 指针率先到达 1 的角色获得行动权，此时时间暂停 。
- 行动与技能：
    - 获得行动权后，角色可选择技能及目标。
    - 伤害公式：最终扣除血量 = 技能伤害值 * (攻击方 ATK / 目标方 DEF) 。
- 行动结束后，该角色进度条指针重置为 0，时间恢复流动。

### 3.3 实体与数值系统 (Entities & Stats)

核心属性： 所有战斗实体（玩家、怪物）均拥有 4 个基础属性：血量 (HP)、攻击 (ATK)、防御 (DEF)、速度 (SPD) 。

### 3.4 胜负与循环判定 (Win/Loss Conditions)

- 单局战斗胜负： 任意一方所有角色 HP 归零即为战斗结束 。
- 怪物战败： 怪物从地图上永久移除，玩家获得奖励，游戏继续 。
- 玩家战败： 玩家被传送回特定的检查点（如本层起点或第一层）重新开始 。
- 最终胜利： 玩家成功抵达塔的顶层 (第 N 层) 。
